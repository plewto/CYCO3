Chords

Chords are defined in one of two ways:

  1. Relative - a list of key-number offsets, i.e. (0 4 7) for a major triad.
  2. Absolute - a list of key-numbers, (60 64 67) for a C-Major triad.

As these are less-then mnemonic there is a global structure called *CHORD-TABLE* which defines about 100 chord-types accessed by name.  Use (?CHORDS) to print a list of available chord-types.

*CHORD-TABLE* basically takes a keyboardist view of chords, there are other possibilities.  The fretworks  plugin  defines chords as they appear on fretted instruments.

TODO Currently, 13 December 2018, fretworks is not completely implemented.  The basic structure is in place and the beginnings of a bass guitar model has been defined.  A 6-string guitar should be defined soon.  At some future point ukulele, banjo and mandolin may be added.

Technically *CHORD-TABLE* and the structures defined by Fretworks are instances of the CHORD-MODEL class.  In use there is no real difference between *CHORD-TABLE* and the fretted chord models.  The types of available chords will be different but otherwise they are used the same way.  *CHORD-TABLE* is the default model for all projects.  Alternate chord-models are selected at either the Section level  or when using a Strummer  Part.

(DEFCHORD name template &optional description)

Defines a new chord for default *CHORD-TABLE*.

name - symbol chord name,  an unofficial style is to name chords inside
square brackets as in [maj].

template - list of keynumber offsets
(?CHORDS &optional (model *chord-table*))

Display chords defined by model.

(CHORD-INVERSION template degree &key add-octave)

Returns inverted form of chord template.

template - list of key offsets.
degree - Sets number and direction of rotations applied to template.
         For positive values the template is rotated to the right degree times, with 
         the left most value transposed up an octave.  Negative values rotate to the
         left.
        
          template '(0 4 7)
          degree --> result
            -2   --> (-8 -5 0)
            -1   --> (-5 0 4)
             0   --> (0 4 7)    no rotation, root position
             1   --> (4 7 12)   1st inversion
             2   --> (7 12 16)  2nd inversion
             3   --> (12 16 19) 
  
:add-octave - Boolean, if true the first note after rotation is repeated 
              at end of template shifted up an octave.  For the above examples 
              with add-octave t

         0 --> (0 4 7 12)
         1 --> (4 7 12 16)
         2 --> (7 12 16 19)
