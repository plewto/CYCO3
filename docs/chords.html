Chords

Chords are implemented as a list of key-number offsets, i.e. the list (0 4 7) is a major triad.  As this is less then mnemonic the global variable *CHORD-TABLE* contains around 100 predefined chord types.

(DEFCHORD name template &optional description)

Defines a new chord for default *CHORD-TABLE*.

name - symbol chord name,  an unofficial style is to name chords inside
square brackets as in [maj].

template - list of keynumber offsets
(?CHORDS &optional (model *chord-table*))

Display chords defined by model.

(CHORD-INVERSION template degree &key add-octave)

Returns inverted form of chord template.

template - list of key offsets.
degree - Sets number and direction of rotations applied to template.
         For positive values the template is rotated to the right degree times, with 
         the left most value transposed up an octave.  Negative values rotate to the
         left.
         
          template '(0 4 7)
          degree --> result
         -2 --> (-8 -5 0)
         -1 --> (-5 0 4)
          0 --> (0 4 7)    no rotation, root position
          1 --> (4 7 12)   1st inversion
          2 --> (7 12 16)  2nd inversion
          3 --> (12 16 19) 
   
:add-octave - Boolean, if true the first note after rotation is repeated 
              at end of template shifted up an octave.  For the above examples 
              with add-octave t

         0 --> (0 4 7 12)
         1 --> (4 7 12 16)
         2 --> (7 12 16 19)
