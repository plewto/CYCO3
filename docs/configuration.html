Configuration Profiles

Profiles are optional but recommended as they customizes CYCO to the local environment. It is possible to use multiple profiles simultaneously as long as they do not conflict.

Profiles are stored as folders under

~/.config/cyco/

Each profile must have a main file, named config.lisp,  which is responsible for loading additional profile files.  Profile names, and all files within a profile directory, MUST  be lower case without embedded white-space. For a profile 'foo' the structure is then:

~/.config/cyco/foo/config.lisp

There are a few functions for loading profile files:

(LOAD-PROFILE &optional name)

Loads the named profile.  If name is not specified default to the previous profile.  The name argument may either be a string or a symbol.  The initial default profile is 'default.

(load-profile 'foo) ;; Loads ~/.config/cyco/foo/config.lisp
(LOAD-SUB-PROFILE name)

Loads named profile without making it the default.  LOAD-SUB-PROFILE is used within a profile to make use of another profile.

(LOAD-PROFILE-FILE filename)

Loads a file relative to the current profile.

(load-profile 'foo)          ;; Makes foo profile current
(load-profile-file ''baz'')  ;; Loads file ~/.config/cyco/foo/baz.lisp

The name and location for configuration fils are specified by the following global variables.

*CYCO-CONFIG-DIRECTORY*

Location of the top level configuration directory. Defaults to  ~/.config/cyco/

*CYCO-CONFIG-FILE*

Name of main profile file, defaults to default.lisp

*CYCO-CONFIG-PROFILE*

Name of current profile, changes with use of LOAD-PROFILE.  defaults to ''default''

Structure of Profile Files

There is none.

Other then the file-name requirements detailed above there is no pre-ordained structure for profile files.  Probably the most common use for profiles, and in fact what they were implemented for, is to describe external MIDI instruments.  In reality you may use them as you wish.

The distribution folder CYCO/config/ contains example profiles:

  • default
  • general-midi
  • sj

general-midi is very basic and used for the example projects.

The 'sj' profile is for my personal work and is much more involved.  It is included for illustration and unlikely to be directly useful to anyone but myself.  It does contain several instrument definitions and I will try to keep it annotated.

The 'default' profile is a sym-link to general-midi.
