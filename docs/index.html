CYCO

CYCO is a lisp based music composition language with a rich set pattern generators and a functional approach to event time specification.

History

CYCO began as a set of extensions to Common Music 2 called CMU (Common Music Utility). CMU primarily added a project level framework and a hierarchical time structure. Eventually Common Music moved to version 3 invalidating much of CMU. As CMU only used a small subset of Common Music's features, a decision was made to separate CMU from Common Music and start from scratch. The name CYCO is a play on the commonly used 'cycle' pattern generator.

CYCO is written in 100% Lisp and was developed and tested with SBCL

As of 11 Septemper 2020, CYCO is known not to work under CLISP due to the STRING-REPLACE function in util/string-utilities.lisp

Features

  • Project centric design.
  • Functional time signatures.
  • A rich set of algorithmic pattern generators.
  • Symbolic names for common music parameters.
  • Chord tables.
  • Hierarchical instrument definitions, IE a snare may be a sub-instrument of a drum-machine.
  • Plugins
  • Under SBCL CYCO's state may be saved as an executable.

Usage overview

Briefly scores are produced by first creating a Project directory to contain all related files. This directory should contain a MIDI sub directory for storing generated MIDI files. The project proper is defined by a series of Lisp files to specify it's structure, an 'orchestra' of instruments, and an arbitrary number of 'Sections.' Once defined the project is rendered to standard MIDI files. Playback requires an external MIDI file player. Many convenience 'shortcut' functions are provided to quickly re-evaluate project files currently under development.

Instalation

Other then a functional Lisp the basic CYCO installation has no external dependencies.

  1. Place cyco3 distribution folder in a convenient location. 
    The default location is ~/dev/cyco3
  2. Create the following directories:
       ~/.config/cyco/
   ~/.config/cyco/plugins/
   ~/cyco-projects/
    If you elect to use alternate locations modify the file cyco3/src/local-config.lisp accordingly.

  3. In a terminal navigate to the cyco3 directory and fire up your favorite Lisp.

  4. At the Lisp prompt enter

    (LOAD "src/cyco")

  5. After the source files load enter the command

    (CYCO)

    At this point you may run the test suite by entering (LOAD "test/cyco-test")
    If you run the test, restart CYCO before proceeding.

  6. CYCO is now ready for use. However you may wish to customize it to match your available MIDI equipment. You do this by creating plugins in the ~/.config/cyco/plugins/ directory as detailed elsewhere.

  7. If you are using SBCL you have the option of saving the current state of CYCO to an executable.

    (SAVE-SNAPSHOT executable-name)

    Snapshots are considerably faster then running from source. After starting CYCO from a snapshot you must first enter (CYCO) at the initial Lisp prompt.
