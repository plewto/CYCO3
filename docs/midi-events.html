MIDI Events

MIDI events are a low-level structure. Typically they are not handled directly while creating a score but they exists just below the surface.


A MIDI Event is defined as a list of form  (time midi-message) .

The following low-level functions return midi-messages.

(MIDI-NOTE-ON  channel keynumber velocity)
(MIDI-NOTE-ON channel keynumber velocity)
(MIDI-POLY-PRESSURE channel keynumber value)
(MIDI-CONTROL-CHANGE channel controller-number value)
(MIDI-CHANNEL-PRESSURE channel value)
(MIDI-PROGRAM-CHANGE channel program-number)
(MIDI-PITCH-BEND channel lsb msb) * see below
(MIDI-SYSTEM-EXCLUSIVE 	data)
(MIDI-END-SYSTEM-EXCLUSIVE)
(MIDI-META-TEXT text)
(MIDI-META-COPYRIGHT text)
(MIDI-META-TRACK-NAME text)
(MIDI-META-INSTRUMENT-NAME text)
(MIDI-META-LYRIC text)
(MIDI-META-CUE text)
(MIDI-META-MARKER text)
(MIDI-TEMPO-CHANGE bpm)
(MIDI-TIME-SIGNATURE num unit)
(MIDI-KEY-SIGNATURE sf &optional (minor nil))

channel is an integer MIDI channel number between 1 and 16 inclusive.
All MIDI data values are integers between 0 and 127 inclusive.  
MIDI pitch bend is specified as two 7-bit bytes.  Two functions are available 
to convert between the lsb and msb bend bytes and a signed-normalized form.

(BEND->MIDI-DATA value)
     Converts bend amount to vector #(lsb msb)
     The bend depth is a singed normalized float -1.0 ≤ value ≤ 1.0

(MIDI-DATA->BEND lsb msb)
     Convert bend lsb and msb bytes to signed normal form.

Event List

An event-list is a sorted list of MIDI events. Events are sorted by time and then by priority. Each MIDI message type has a priority which determines the sort order when they occur at the same time. The most common example are note-on and note-off events. If they have the same time value the note-off event is sorted after the note-on. This is a low-level detail handled behind the scenes by the SORT-MIDI-EVENTS function.
