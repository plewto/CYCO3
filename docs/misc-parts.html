Miscellaneous Part Types

This page details several specialized part  types.

  • Raw-Part
  • Metronome
  • Programs

Programs

Programs is a type of part  for creating program-change-events.

(MAKE-PROGRAMS name instruments &key time section remarks render-once)
(PROGRAMS name instruments &key time section remarks render-once)

MAKE-PROGRAMS and PROGRAMS are nearly identical.  The only difference is that PROGRAMS binds the new part object to a symbol with the same name while MAKE-PROGRAMS does not.  The following two lines are equivalent

(param foo (make-programs 'foo ...))  ;; name must be quoted symbol
(programs foo ...)                    ;; name must -not- be quoted.

A Programs part is often placed at the start of a Section.

Arguments:

name         - Symbol, part name
instruments  - List of instruments
:time        - Time of events should be in format expected by cue function,  default nil.
:section     - Defaults to current section of *PROJECT*.
:render-once - Boolean, if true events are only generated once.
:remarks     - Optional text.


Example usage:	 

(programs foo-programs (list bass piano organ))

Metronome

(MAKE-METRONOME &key ...)
(METRONOME &key ...)

MAKE-METRONOME and METRONOME both create a metronome part. The only difference is that METRONOME binds the new part to a symbol with the same name while MAKE-METRONOME does not. The default name is  <SECTION-NAME>-METRONOME

Keyword arguments

:NAME       - Defaults to 'section-metronome
:SECTION    - Defaults to current section of *PROJECT*
:CUEFN      - Inherits from section. 
:TEMPO      - Inherits from section.
:UNIT       - Inherits from section.
:BARS       - Inherits from section.
:BEATS      - Inherits from section.
:CUE        - Time cue pattern, default creates an event on every beat.
:KEY        - Keynumber pattern
:AMP        - Amplitude pattern
:INSTRUMENT - Defaults to *METRONOME*

The *METRONOME* instrument uses specialized keynumber  and dynamic  maps to recognize the following symbols:

  • :PHRASE - occurs on the first beat of the first bar.
  • :BAR - occurs on the first beat of every bar except the first bar.
  • :BEAT - occurs on all other beats.

The *METRONOME* instrument is not automatically defined.  Normally it is established as part of the configuration  process.

RAW-PART

A raw-part is a primitive type of part  intended as a fall-back when no other part does the job.  Raw-parts are for edge cases and generally you will want to use some other part type.

(MAKE-RAW-PART name &keys ...)
(RAW-PART name &keys ...)

Creates new raw-part, the only difference between MAKE-RAW-PART and RAW-PART is that the later binds the new part to a symbol with then same name. The following two lines are equivalent:

(param foo (make-raw-part 'foo ...))  ;; name must be quoted symbol
(raw-part foo ...)	              ;; name must -not- be quoted

Keywords

:section      - defaults to current section of *PROJECT*
:bars         - bars per phrase, inherits from section.
:beats        - beats per bar, inherits from section
:render-once  - boolean, if true do not repeat.
:transposable - boolean if true this part is subject to transpose and
                key invert operations.  Raw-parts may not be reversed
                by retrograde operations.
:remarks      - Optional remarks text.
:events       - Nested list of events.

Event times are specified in absolute seconds from the start of the parent section.  A Raw-Part does not use the time-signature or a cue-function to determine event times, you are on your own.

Events are specified as a cons (time . midi-message)

((time-1 . midi-message-1)
  (time-2 . midi-message-2)
   .......................
  (time-n . midi-message-n))

See MIDI Events

MIDI system-exclusive and meta messages are one area when raw-parts could be useful.  Currently (2018.11.18) meta-messages are used internally in a few places and there are no higher-level system-exclusive functions, though some may be added later.
