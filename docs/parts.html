Parts

A Part  is an extension of time-signature  which combines a set of instruments  with instructions for what they are to play.  There are several types of Part and they form the bulk of a project's code.

Parts are always used within a section,  from which they inherit default timing parameters.

Transformations

Some Part types are subject to transpose, key-inversion and retrograde operations. If a part's TRANSPOSABLE property is false then it is not effected by either the transpose or invert functions. If the parts REVERSIBLE property is false, then it is not effected by the retrograde function.

Part Repetitions

Each part is repeated as needed to fill the section duration, i.e. a part of duration 4 bars in an 8-bar section is repeated twice. If the part duration does not evenly divide the section duration, then the repetition count is truncated. For example a 3-bar part in an 8-bar section is repeated twice. There is no partial repetition; it is not repeated 2 2/3 times.

Things get more complex if the part time-signature does not match the section's. These situations have not been well tested. You can expect the relative times between the part and section to go in and out of phase. I believe all part repetitions repeat the part in it's entirety, there is no final partial part.

There is one property that does effect part repetition. If the RENDER-ONCE property is true, the part is produced one time only, regardless of it's bar-count.
