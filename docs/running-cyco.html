Running CYCO

CYCO may either be ran from source, or if using SBCL, a saved snapshot.  This illustration assumes SBCL and uses the following symbols:


#     - shell comment
$     - command line prompt.
*     - lisp REPL prompt.
CYCO: - lisp REPL prompt while using CYCO (changes to reflect the current project name)

To run CYCO from source:

cd into the CYCO distribution directory.

    $ cd ~/CYCO

Start your favorite lisp and load the source.

    $ sbcl --load src/cyco

After the source files load switch to the CYCO package.

    * (CYCO)

The lisp prompt should change to CYCO:  As projects are created the 
prompt text changes to reflect the current project name.

After CYCO has loaded you'll want to configure it to match your local MIDI environment.  This is accomplished by writing plugins.  Initially you may use the predefined General MIDI  plugin

CYCO: (plugin 'general-midi)

Once configured you may save a snapshot executable.  Snapshots have the advantage of 'baking-in' the current state of CYCO for future use.  They also have near instantaneous startup time. Use the following command to create a snapshot in your home directory.

CYCO: (save-snapshot "~/cyco-snapshot-name")
