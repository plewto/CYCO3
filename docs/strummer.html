Strummer

A STRUMMER  is a PART with heavy emphasis on producing chords.
Strummer features:

  • Alternate chord models
  • Chord inversions
  • Strum chords with acceleration and direction patterns.
  • Grace Notes
  • Amplitude scale notes across a chord.
  • Amplitude patterns including crescendos.
  • Amplitude randomization.
  • Simple events for bend, MIDI controller and program changes.
(MAKE-STRUMMER name instrument &key ...)
(STRUMMER name instrument &key ...)

Create Strummer part. MAKE-STRUMMER and STRUMMER are identical except that strummer binds the new object to a symbol with the same name. The following two lines are equivalent.

(param foo (make-strummer 'foo instrument ...))   ;; name must be quoted
(strummer foo instrument ...) 	                  ;; name must -not- be quoted

Keyword options:

:section      - Parent section, defaults to current section of *PROJECT*
:cuefn        - Cueinig function, defaults to section value.
:shift        - Initial offset time (ISSUE: Verify shift is implemented)
:tempo        - Default inherited from section.
:unit         - Default inherited from section.
:bars         - Default inherited from section.
:beats        - Default inherited from section.
:subbeats     - Default inherited from section.
:render-once  - Boolean, if true do not repeat events when rendered.
:transposable - Boolean, if false transpose and invert methods have no effect.
:reversible   - Boolean, if false retrograde method has no effect.
:chord-model  - Default inherited from section.
:remarks      - Optional remarks text.
:events       - List, see below.

Strummer events are specified as a nested list of clauses  where each clause begins with a keyword and then zero or more arguments.  The exact number of arguments is dependent of the specific command.  Commands fall into one of three categories:

  1. Test
    :TRACE :BREAK
  2. Parameter values. Most of the commands set a parameter value, such as note amplitude or duration.
    These do not produce any events and remain in effect until explicitly changed.
    :RESET :TIME :CHORD :INVERSION :OCTAVE :STRUM :STRUM* :DIRECTION
    :AMP* :END-TOGETHER :GRACE-AMP* :GRACE-DURATION :GRACE-DELAY
    :DUR :AMP :CRES :AMP-BLUR :AMP-LIMITS
  3. Generators, these commands produce one or more MIDI events.
    :KEY :GRACE :BEND :CC :PROGRAM :BANK
:TRACE-OFF
:TRACE-ON

Turn diagnostic tracing on/off.

:BREAK

Skip evaluation of remainder of events

:RESET

Returns all parameters to default values.

:TIME (list xxx)

Establish time for following events.  The exact format is dependent on the cue-function.

:CHORD name
:CHORD (template-list)

Specifies chord, either by name or as a list of key-number offsets.  A list of offsets is always relative to the current key-number, i.e. (0 4 7) is a major triad.  Named chords must be defined by the chord-model in use.  Use the function (nobr>(?chord-mode model) to display list of available chords.  By convention chord names are enclosed in square brackets, as in [maj], though there are exceptions.  Named chords may either be relative to the key-number, as above, or they may be a list of absolute key-numbers.  depending on the chord-model.  The default model, *CHORD-TABLE*, uses relative chords.  Chord models provided by the fretworks plugin use absolute chords.

To effectively disable chords use (0).

:INVERSION n

Sets chord inversion.

:OCTAVE n

Adds octave copy of initial note to chord template.

:STRUM delay

Sets delay time between individual chord notes.&nobr; The delay may either be in absolute seconds, or a relative metric expression.

:strum 0.01     ;; produce chord notes 1/100 second apart, ignores tempo.
:strum s        ;; produce chord notes with sixteenth note delay, sensitive to tempo.
:STRUM* s

Adds acceleration to strummed notes.

0.125 ≤ s ≤ 8.00, default 1.0 = no acceleration.
:DIRECTION d
:DIRECTION list

Sets strum direction pattern.  Possible values are

  down   - play notes in order
  up     - reverse order
  dice   - select up or down randomly
  random - play notes in random order

A list it is treated as a cycle pattern applied over future key events.

:AMP* s

Applies amplitude scaling to chord notes.  The first note is played at the nominal amplitude as0, the second note is scaled to as1, the third to as2, etc.

0.25 ≤ s ≤ 4.0
:END-TOGETHER no
:END-TOGETHER yes

Selects how chord note durations are handled.  If the argument is no, the note endings are staggered in the same order they are played.  If the argument is yes, all notes are held to same duration.

:GRACE-AMP* s

Grace note amplitude relative to nominal amplitude.

0.25 ≤ s ≤ 4.0
:GRACE-DURATION d

Grace note duration. Argument may be in absolute seconds or a metric expression.

:GRACE-DELAY dly

Grace note onset delay relative to current time. Argument may be absolute time in seconds or a metric expression.

:DUR d

Note duration. Argument may be absolute time in seconds or a metric expression.

:AMP a
:AMP list

Sets note amplitude pattern. If the argument is a list, it is treated as a cycle pattern and applied to future key events.

:CRES start end count

Sets up crescendo/decrescendo over the next count notes.  Once final amplitude has been reached it remains in effect until explicitly changed.

count - number of steps  0 < count
start - initial amplitude
end   - final amplitude.
:AMP-BLUR s

Amplitude blurring ratio.

0.0 ≤ s ≤ 1.0
A value of 0.0 disables blurring
A value of 0.5 indicates nominal amplitude may be changed by up to 50%.
:AMP-LIMIT min max

Sets limit on minimum and maximum amplitudes.
Amplitude related parameters (:AMP :AMP* :GRACE-AMP* :AMP-BLUR and :CRESS) are not allowed to exceed these limits.

:KEY k

Generate note event.  key-number k is first processed by the instruments keynumber-map.

The actual events generated is determined by current chord, strum, amplitude and duration parameters.

:GRACE k

Produce a grace note.  The key-number k is first processed by the instruments keynumber-map.

Grace notes are single-events only which ignore chord and strum parameters.

:BEND b

Produce a single MIDI pitch-bend event.  Use a Controllers part for more complex options.

-1.0 ≤ b ≤ +1.0
:CC ctrl value

Produce a single MIDI controller event.  Use a Controllers part for more complex options.

ctrl - integer controller number  0,1,2,...,127
value - normalized value  0 ≤ value ≤ 1.0
:PROGRAM prg
:BANK bnk prg

Produce program-change events.  The exact format for program prg and bank bnk arguments is dependent on the instrument's program-map.  in either case the symbol 'default selects the default program and/or bank.  Most program-maps should treat integer arguments as MIDI program numbers.
