Time Signatures

A time-signature  is a type of cyco-node which sets basic timing parameters. All composition related classes inherit from time-signature. This allows the default, project-level, time signature to be superseded by more specific classes. Jumping ahead a bit the project class hierarchy is as follows:


CYCO-NODE
 |
 +-- TIME-SIGNATURE
      |
      +-- PROJECT
      |
      +-- SECTION
      |
      +-- PART

A project  is composed of sequential sections,  which are composed of parts.  Each section automatically inherits time-signature parameters from the project, and each part in turn inherits from their parent section. At any level one or more time-signature parameters may be overridden.

Time Signature Parameters

The time-signature class extends the normal concept of a time-signature by including the following parameters.

  • Tempo - in BPM
  • Beat unit - quarter note by default.
  • Bar count - number of bars to a phrase.
  • Beat count - number of beats per bar.
  • Subbeat count - number of 'subbeats' per beat. The default value of 4 sets 4 sixteenth notes per quarter note.
  • Tbeats - number of alternate subbeats per beat. The default value of 3/2 sets 6 sixteenth note triplets per beat.
(TEMPO tsig)
(UNIT tsig)
(BARS tsig)
(BEATS tsig)
(TBEATS tsig)
(SUBBEATS tsig)
(TSUBBEATS tsig)
(TICKS-PER-BEAT tsig)
(TICKS-PER-SUBBEAT tsig)

The above 'getter' methods return the indicated time-signature parameter.

(TEMPO! tsig value)
(UNIT! tsig value)
(BARS! tsig value)
(BEATS! tsig value)
(SUBBEATS! tsig value)

The above 'setter' methods updates the indicated parameters.

(PHRASE-DURATION tsig)
(BAR-DURATION tsig)
(BEAT-DURATION tsig)
(SUBBEAT-DURATION tsig)
(TBEAT-DURATION tsig)
(TSUBBEAT-DURATION tsig)
(TICK-DURATION tsig)

The above methods returns the duration of the indicated time-signature parameter.
